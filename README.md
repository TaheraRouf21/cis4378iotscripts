--- IoT Honeypot Simulation Project ---
Overview:
  This project simulated IoT enviornment with a deployed honeypot designed to detect, log, and analyze malicious activity. The goal is to better understand how attackers target vulnerable IoT devices and to demonstrate basic defensive security monitoring techniques. 
  This project was built as part of a cybersecurity learning experience focused on network security, threat detection, and system monitoring.

Objectives:
- Simulate a vulnerable IoT device environment
- Deploy a honeypot to attract malicious traffic
- Monitor and log attacker behavior
- Analyze common attack patterns and techniques
- Demonstrate security concepts relevant to real-world systems

Technologies Used:
- Kali Linux- virtual machine
- Python- for scripting and logging
- Network monitoring tools- netstat/ ss
- SSH/ Telnet services- for simulation
- Basic Linux proceess monitoring tools

Project Architecture- The simulation consists of:
- A virtual environment acting s the IoT device
- Exposed services to mimic vulnerable endpoints
- A honeypot configured to log incoming connections
- Monitoring tools to track system and network activity
✨✨✨✨
SETUP INSTRUCTIONS:
1. Environment Setup:
   - Install VirtualBox
   - Set up Kali LInux virtual machine
   - Ensure internet/network access is enabled
     
2. Install Required tools:
   - Update your system using: sudo apt update && sudo apt upgrade -y
   - Install necessary tools: sudo apt install net-tools openssh-server python3 -y
  
3. Configure the Honeypot
   - Enable SSH service: sudo service ssh start
   - Create a Python script to log incoming connections OR use a simple logging tool
  
4. Monitor Network Activity:
   - Use commands like: ss -tulnp OR netstat -tulnp
   - Look for unknown IP connections, repeated login attempts, suspicious open ports

5. Simulate Attacks
   - Use another machine or tool to attempt login- brute force
   - Observe logs generated by the honeypot
✨✨✨✨

Observations/ Results- during the simulation, the following behhaviours were observed:
 - Repeated connection attempts on open ports
 - Unathorized login attempts
 - Patterns consistent with automated scanning tools
These findings demonstrate how easily exposed IoT devices can become targets.

Key Takeaways:
- IoT devices are highly vulnerable when improperly secured
- Weak credentials are common attack vectors
- Monitoring tools can help detect suspicious behavior early
- Honeypots are useful for understanding attacker techniques

Limitations:
- This is a simulated enviornment, not a production system
- Limited scale compared to real-world IoT networks
- Basic logging, no advacned SIEM integration
